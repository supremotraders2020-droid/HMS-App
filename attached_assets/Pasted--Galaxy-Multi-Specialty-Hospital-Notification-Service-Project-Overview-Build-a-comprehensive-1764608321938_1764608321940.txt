# Galaxy Multi Specialty Hospital - Notification Service

## Project Overview

Build a comprehensive hospital notification service for **"Galaxy Multi Specialty Hospital"** featuring multi-channel notifications (push, email, SMS, WhatsApp), hospital team management, and an attractive responsive interface with modern medical-grade design.

**Hospital Address:** Sane Chowk, Nair Colony, More Vasti, Chikhali, Pimpri-Chinchwad, Maharashtra 411062

---

## Core Features

### 1. Multi-Channel Notification System

Create a notification management system that supports multiple communication channels:

- **Push Notifications** - In-app and browser push notifications
- **Email Notifications** - Using SendGrid integration for professional email delivery
- **SMS Notifications** - Text message alerts for urgent communications
- **WhatsApp Notifications** - WhatsApp Business API integration for patient communication

#### Notification Categories
- Health Tips
- Hospital Updates
- Emergency Alerts
- OPD Announcements
- Disease Alerts
- General Notifications

#### Priority Levels
- Low (routine updates)
- Medium (important notices)
- High (urgent matters)
- Critical (emergency alerts with visual pulse effects)

---

### 2. Advanced Notification Form Wizard

Implement a 2-step form wizard for creating notifications:

#### Step 1: Compose Message
- Title input with character limit
- Rich text message composition
- Category selection with icons
- Priority level selection with visual indicators
- Scheduled delivery option with date/time picker

#### Step 2: Channels & Media
- Multi-channel selection (Push, Email, SMS, WhatsApp)
- Live preview for each selected channel
- Drag-and-drop file upload with object storage integration
- Link attachment capability
- Media file validation (10MB limit, allowed types: images, videos, PDFs)

#### Form Features
- Progress indicator showing current step
- Live preview of notification appearance
- Form validation with clear error messages
- Draft saving capability
- Success/error feedback with toast notifications

## Data Models

### Notification Schema
```typescript
{
  id: number
  title: string
  message: string
  category: string
  priority: 'low' | 'medium' | 'high' | 'critical'
  channels: string[]  // ['push', 'email', 'sms', 'whatsapp']
  scheduledAt: Date | null
  mediaFiles: MediaFile[]
  status: 'draft' | 'scheduled' | 'sent' | 'failed'
  createdAt: Date
  createdBy: string
}
```

### Team Member Schema
```typescript
{
  id: number
  name: string
  title: string

  specialization: string
```


## API Endpoints

### Notifications
- `GET /api/notifications` - List all notifications
- `POST /api/notifications` - Create new notification
- `GET /api/notifications/:id` - Get notification details
- `PUT /api/notifications/:id` - Update notification
- `DELETE /api/notifications/:id` - Delete notification

### Team Members
- `GET /api/team-members` - List all team members
- `POST /api/team-members` - Add new team member
- `GET /api/team-members/:id` - Get member details
- `PUT /api/team-members/:id` - Update member
- `DELETE /api/team-members/:id` - Remove member

### File Uploads
- `POST /api/uploads` - Upload files to object storage
